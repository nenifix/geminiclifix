3. STEP-BY-STEP DEPLOYMENT GUIDES
================================================================================

──────────────────────────────────────────────────────────────────────────────
OPTION A: Oracle Cloud (Best Free VPS) — Ubuntu 22.04
──────────────────────────────────────────────────────────────────────────────

1. SIGN UP
   - Go to https://cloud.oracle.com/free
   - Create account (need credit card for verification, won't be charged)
   - Select region closest to you

2. CREATE INSTANCE
   - Click "Create a VM"
   - Name: nenicoder
   - Image: Ubuntu 22.04
   - Shape: VM.Standard.E2.1.Micro (always free) OR
           VM.Standard.A1.Flex (ARM, 4 OCPU, 24GB RAM — also free!)
   - SSH key: Generate new key pair, download private key
   - Click Create

3. CONNECT VIA SSH
   ssh -i ~/.ssh/your-key ubuntu@<PUBLIC_IP>

4. INSTALL NODE.JS
   curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
   sudo apt-get install -y nodejs git

5. CLONE & SETUP
   git clone https://github.com/nenifix/nenicoder.git
   cd nenicoder
   npm install
   npm run build

6. CONFIGURE .env
   nano .env
   # Add your TELEGRAM_BOT_TOKEN, ALLOWED_USER_ID, OPENROUTER_API_KEY, MODEL

7. INSTALL PM2 (process manager)
   sudo npm install -g pm2
   pm2 start dist/index.js --name nenicoder
   pm2 save
   pm2 startup

8. OPEN FIREWALL (if needed)
   sudo ufw allow 3001/tcp  # For REST API
   sudo ufw enable

9. DONE! Bot runs 24/7. Check with: pm2 logs nenicoder

──────────────────────────────────────────────────────────────────────────────
OPTION B: Railway (Easiest — No Server Management)
──────────────────────────────────────────────────────────────────────────────

1. SIGN UP at https://railway.app (login with GitHub)

2. CREATE NEW PROJECT
   - Click "New Project"
   - Select "Deploy from GitHub repo"
   - Authorize GitHub, select nenifix/nenicoder

3. ADD ENVIRONMENT VARIABLES
   - Click on your service → Variables
   - Add:
     TELEGRAM_BOT_TOKEN = your_token
     ALLOWED_USER_ID = your_user_id
     OPENROUTER_API_KEY = your_key
     MODEL = openrouter/anthropic/claude-sonnet-4

4. CONFIGURE START COMMAND
   - Settings → Start Command: npm start
   - Or add a Dockerfile (see below)

5. DEPLOY
   - Railway auto-deploys on git push
   - Get URL from Settings → Networking

──────────────────────────────────────────────────────────────────────────────
OPTION C: Render (Free Tier)
──────────────────────────────────────────────────────────────────────────────

1. SIGN UP at https://render.com

2. CREATE NEW WEB SERVICE
   - Connect GitHub repo: nenifix/nenicoder
   - Runtime: Node
   - Build Command: npm install && npm run build
   - Start Command: npm start

3. ADD ENVIRONMENT VARIABLES (same as above)

4. DEPLOY — auto-deploys on git push

──────────────────────────────────────────────────────────────────────────────
OPTION D: Docker Deployment (Any VPS)
──────────────────────────────────────────────────────────────────────────────

Create Dockerfile in project root:

  FROM node:20-alpine
  WORKDIR /app
  COPY package*.json ./
  RUN npm ci --only=production
  COPY . .
  RUN npm run build
  EXPOSE 3001
  CMD ["node", "dist/index.js"]

Build & run:
  docker build -t nenicoder .
  docker run -d --name nenicoder --restart always \
    -e TELEGRAM_BOT_TOKEN=xxx \
    -e ALLOWED_USER_ID=xxx \
    -e OPENROUTER_API_KEY=xxx \
    -e MODEL=openrouter/anthropic/claude-sonnet-4 \
    nenicoder

──────────────────────────────────────────────────────────────────────────────
OPTION E: Fly.io (Docker-based, Free)
──────────────────────────────────────────────────────────────────────────────

1. INSTALL flyctl
   curl -L https://fly.io/install.sh | sh

2. SIGN UP
   fly auth signup

3. LAUNCH
   cd nenicoder
   fly launch --name nenicoder

4. SET SECRETS
   fly secrets set TELEGRAM_BOT_TOKEN=xxx
   fly secrets set ALLOWED_USER_ID=xxx
   fly secrets set OPENROUTER_API_KEY=xxx
   fly secrets set MODEL=openrouter/anthropic/claude-sonnet-4

5. DEPLOY
   fly deploy

4. KEEPING IT RUNNING 24/7
================================================================================

PM2 (Recommended for VPS):
  npm install -g pm2
  pm2 start dist/index.js --name nenicoder
  pm2 save          # Save process list
  pm2 startup       # Auto-start on reboot
  pm2 logs          # View logs
  pm2 restart nenicoder  # Restart
  pm2 stop nenicoder     # Stop

Systemd (Alternative):
  sudo nano /etc/systemd/system/nenicoder.service

  [Unit]
  Description=NeniCoder Telegram Bot
  After=network.target

  [Service]
  Type=simple
  User=ubuntu
  WorkingDirectory=/home/ubuntu/nenicoder
  ExecStart=/usr/bin/node dist/index.js
  Restart=always
  RestartSec=10
  Environment=NODE_ENV=production

  [Install]
  WantedBy=multi-user.target

  sudo systemctl enable nenicoder
  sudo systemctl start nenicoder
  sudo systemctl status nenicoder

================================================================================
TROUBLESHOOTING
================================================================================

Bot not responding?
  1. Check logs: pm2 logs nenicoder
  2. Verify .env: cat .env
  3. Test manually: node dist/index.js
  4. Check firewall: sudo ufw status

Out of memory?
  - Use swap: sudo fallocate -l 1G /swapfile && sudo chmod 600 /swapfile && sudo mkswap /swapfile && sudo swapon /swapfile
  - Or upgrade to Oracle ARM (24GB RAM free)

Port already in use?
  - Kill old process: pm2 delete nenicoder
  - Or: lsof -i :3001 && kill -9 <PID>

GitHub push not deploying?
  - Check webhook: Settings → Webhooks in GitHub repo
  - Manual deploy: git push origin master
